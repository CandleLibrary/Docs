Scopes

[ <scope> | <ws> ]

The scope elements is the heart and soul of a wick component. It defines data flow boundaries and provides mechanisms to integrate scripts and CSS code into a scoped set of elements. Every component that wick generates has a scope assigned to it's root element.

Data Flow

Data flow between scopes is strictly limited. Any data between components is blocked at <scope> tags. In the below example, there are two child scopes that have bindings for `name`. Input in the lower scope will send data through the `name` binding, but only the `name` text binding in the lower scope will receive the data. The content of the upper scope will remain unchanged. Transitions

Containers provide transition messages that allow you to create animations depending on whether an element is entering or leaving the container, or if its index position changes.
  • trs_asc_in: The element is transitioning into view, from a lower index position into a higher index. i.e in from the top of the screen
  • trs_asc_out: The element is transitioning out of the view, from a lower index position into a higher index. i.e. out through the bottom of the screen
  • trs_dec_in: The element is transitioning into view, from a higher index position into a lower index. i.e. in form the bottom of the screen
  • trs_dec_out: The element is transitioning out of the view, from a higher index position into a lower index. i.e. out through the top of the screen
